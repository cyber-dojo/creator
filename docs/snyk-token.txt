
main.yml contains a job security-scan which contains calls to snyk, eg

    - snyk container test ${IMAGE_NAME} ...

This relies on the SNYK_TOKEN masked env-var, which is a cyber-dojo Org level Gitlab secret.
See https://gitlab.com/groups/cyber-dojo/-/settings/ci_cd#ci-variables

It is currently set to the AuthToken of the snyk user/organization JonJagger
See https://app.snyk.io/account
Account settings 
  General

Note that in Gitlab, all env-vars are automatically available in workflow jobs;
they do not have to be explicitly added to the workflow job like they do in GitHub.
