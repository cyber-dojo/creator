The main src code is in the /app dir.
This is added to the docker image using the Dockerfile command:
COPY --chown=nobody:nogroup . /app
Note: this sets the ownership of everything underneath /app
but it does not set the ownership of /app itself.

There are some tests in /app/test_server
These are currently volume-mounted into, and run from
inside a docker container using docker-compose.yml:

  creator-server:
    read_only: true
    volumes: [ "./app/test_server:/app/test_server:ro" ]

I would prefer to have code and tests in a non parental dir structure. Eg
/app
/test

See runner for a way to do this...
runner/test/coverage.rb
  require 'simplecov'
  SimpleCov.start do
    filters.clear # <<<<<<<<<<<<
    add_group('app') { |src| src.filename =~ %r"^/app/" }
    add_group('test') { |src| src.filename =~ %r"^/test/.*_test\.rb$" }
  end
  SimpleCov.root('/app') # <<<<<<<<<<<
  SimpleCov.coverage_dir(ENV['COVERAGE_ROOT'])
